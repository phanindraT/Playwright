npx playwright test :  Runs all test files in the tests folder on all configured browsers (Chromium, Firefox, WebKit) by default.

npx playwright test --ui : Opens the Playwright Test UI for selecting and running tests interactively.

npx playwright test mytest.spec.ts : Runs only the specified test file.

npx playwright test mytest.spec.ts mytest2.spec.ts : Runs the specified test files.

npx playwright test -g verify : Executes tests where the test title includes the word "verify".

npx playwright test -g "Login.*Success" : Runs all tests where the title matches the regex pattern "Login" followed by any text and then "Success".​

npx playwright test mytest.spec.ts --project=chromium : Runs tests only on the Chromium browser. Replace chromium with firefox or webkit as needed.

npx playwright test --headed : Runs browsers with the GUI visible.

npx playwright show-report : Opens the Playwright HTML test report.

example : Using Regex for URL Assertions & Pattern Matching
await expect(page).toHaveURL(/automation-practice/);  --> This passes if the URL contains "automation-practice" anywhere in it.

await expect(page).toHaveURL(/https:\/\/example\.com\/dashboard.*/); --> This matches URLs starting with https://example.com/dashboard.

Playwright's distinct prioritization of locators:
First Priority: Playwright built-in locators (getByRole, getByText, getByLabel, etc.).
Second Priority: CSS selectors.
Third Priority: XPath selectors.

DOM Concept :Locators interact with DOM elements — not directly with the raw HTML or UI — but with the browser-generated DOM. API interfrace provided by browser.
Built-In Locator Methods: page.getByRole(), page.getByText(), page.getByLabel(), page.getByPlaceholder(), page.getByAltText() (for images with alt attributes),
page.getByTitle(), page.getByTestId().

Assertions use the expect() function, commonly paired with checks such as toBeVisible() for visibility, or toHaveText() for text validation.

** Use await where actions or assertions return promises or perform actions.
Don't use await for pure locator retrieval (unless the method is asynchronous). --> In Playwright with TypeScript, locators represent references to elements on the page and are designed to be lightweight and lazily evaluated.
This means: Locator retrieval methods like page.locator() or built-in locators (page.getByText(), page.getByAltText(), etc.) do not return promises and do not require await. They simply create and return a locator object that can later be used to interact with the element(s).
Use await only when you are performing an action on the locator or asserting something that returns a promise. For example, when you do something like:
await locator.click()
await expect(locator).toBeVisible()
await locator.fill('text')
Because these above methods perform asynchronous operations (interacting with the browser or waiting for conditions), they return promises and must be awaited.

const btn = page.locator('button')	 --> No await required.
